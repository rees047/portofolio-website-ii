POKEMON

Objective: The objective of this project is to display the capability of html, css and boostrap and jquery in simple projects while using an external API to get data.

Featured Techonologies: HTML, CSS, JQuery

User Story:
As a user,
I want to click on the pokeball
so that I can see data pokemon data

Acceptance Criteria:
Given the user has chosen desired pokeball to click
When the user clicks on the pokeball
Then the pokeball opens and shows data about chosen pokemon

CHAT APP

Objective: This is a chat-app project created with native-react-app this is to demonstrate the difference of web browsers between native mobile devices.
Expo Go is the app used to test out the chat-app application in mobile devices. Expo Go can be downloaded from google play Expo instructions can be found below
This app also uses Google's Firebase Firestore to store data.

Featured Techonologies: Native React, Expo Go, Google Firebase, Gifted Chat App

User Story:
As a user,
I want to be able to login
so that I can send a chat message

Acceptance Criteria:
Given the user is logged in
When the user sends a message
Then the chat is sent and received by the other user

Github Repository: Movie Client

Live Demo: There is no live demo for this app. You need to install Expo Go on your phone and/or Android Studio (if you're doing it on a desktop) to be able to run and test the app.

MEET MY CITY

Objective: The objective of this project is to build a serverless, progressive web application (PWA) with React using a test-driven development (TDD) technique. This application uses the google Calendar API to fetch upcoming events.

Featured Techonologies: React

User Story:
As a user,
I want to choose a city
so that I can see the upcoming events within that city

Acceptance Criteria:
Given the user types in a city
When the user chooses a city from the suggestion box
Then events related to the chosen city are shown

MOVIE API

Objective: To demonstrate the capability of NODE JS. The movie api is built using mongodb as the database and authentication is implemented by passport jwt, json webtoken and following CORS policy.
Endpoints are used to access necessary data from the API. The API endpoints are tested through Postman.

Featured Techonologies: NodeJS, MongoDB, Postman

User Story:
As a user,
I want to be request data through API endpoints
so that I can see get corresponding results

Acceptance Criteria:
Given the user wishes for all movie list
When the user requests data using All Movie List Endpoint
Then a list of movies are returned
